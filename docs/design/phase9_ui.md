# Phase 9 설계: UI 고도화 (승인 대기 중)

## 배경

과제 명세서의 예시 UI 화면(메인 메뉴에 시스템 현황 요약, 모니터링에 잔여율 바)을 아직 반영하지
않았다. 지금까지는 각 메뉴가 개별 표/메시지만 보여줬는데, 한눈에 전체 상태를 파악할 수 있도록
개선한다.

## 변경 범위

### 1. 메인 메뉴 - 시스템 현황 요약
메뉴 목록 위에 아래와 같은 요약 정보를 표시한다(과제 명세 예시 UI와 동일한 구성).
```
시스템 현황
등록 시료   2종     총 재고     304 ea
전체 주문   3건     생산라인    1건 대기
```
- 등록 시료 종수 = `SampleController.list_all()` 길이
- 총 재고 = 모든 시료 재고 합
- 전체 주문 건수 = `OrderController.list_all()` 길이 (REJECTED 포함, "전체"이므로)
- 생산라인 대기 건수 = `ProductionController.list_queue()` 길이

### 2. 모니터링 - 재고 잔여율 바
재고량 확인 화면에 텍스트 기반 막대(잔여율 %)를 추가한다. 기준치는 200ea를 "가득 참(100%)"으로
잡아 시각적 비교가 되게 한다(임계값은 CLAUDE.md/PoC 기준과 별개로 표시용).
```
ID      이름              재고   상태   잔여율
S-001   실리콘 웨이퍼-8인치  280   여유   [##########] 100%
S-003   SiC 파워기판-6인치   24   부족   [##........] 12%
```
- 잔여율 = `min(100, round(재고 / 200 * 100))`, 막대는 10칸 기준으로 `#`/`.`로 표시

### 3. 상태 표기 통일
주문 목록(`show_orders`)의 상태 컬럼을 대괄호로 감싸 `[RESERVED]` 형태로 표기해 가독성을 높인다
(ANSI 컬러는 사용하지 않음 - 터미널 호환성 및 자동 테스트 안정성 고려).

## 영향 범위

- `app/views/console_view.py`: 요약/막대/상태 표기 함수 추가·수정
- `app/controllers/main_controller.py`: 메인 메뉴 표시 시 요약 데이터 조립
- 기존 Controller/Repository 로직은 변경 없음 (순수 View 계층 개선)

## 테스트 전략

- View는 입출력 전담이라 로직 테스트 가치가 낮으므로, "요약 데이터 조립"과 "잔여율 계산"처럼
  숫자를 다루는 부분만 별도 순수 함수로 분리하여 pytest로 검증한다.
  - `summarize_dashboard(samples, orders, queue) -> dict` (총 재고/건수 계산)
  - `stock_bar(stock: int, capacity: int = 200) -> str` (잔여율/막대 문자열 생성)
- 나머지 출력 포맷은 콘솔 E2E 수동 검증으로 확인한다.

## 검토 필요 사항

1. 잔여율 기준치 200ea가 적절한지, 다른 값이나 시료별 개별 기준(예: 초기 재고 대비 %)을 원하시는지
2. 전체 주문 건수에 REJECTED를 포함할지(제안: 포함 - "전체"라는 의미를 살리기 위해), 제외할지
