# Phase 5 설계: 생산 라인

## 목표

생산 큐에 등록된 부족분을 FIFO로 처리하여 실 생산량/생산 시간을 계산하고,
생산 완료 시 주문 상태를 PRODUCING → CONFIRMED로 전환하며 재고를 반영한다.
생산 현황(진행 중 항목)과 대기열을 조회하는 기능도 포함한다.

## 계산 규칙 (PRD 기준)

- 실 생산량 = `ceil(부족분 / 수율)`
- 총 생산 시간 = `평균 생산시간 * 실 생산량`

## 아키텍처

```
app/controllers/production_controller.py
```

## 동작 규칙

- `process_next()`: 큐의 맨 앞(FIFO) 항목을 꺼내 실 생산량/생산 시간을 계산하고,
  해당 시료의 재고에 (실 생산량 - 부족분)만큼 순증가시켜 결과적으로 주문 수량만큼 재고가 확보되도록 한다.
  이후 해당 주문을 CONFIRMED로 전환한다.
  - 예: 부족분 170, 수율 0.92 → 실 생산량 = ceil(170/0.92) = 185. 재고는 185만큼 생산되어 필요한 170은
    출고 대기(CONFIRMED)로 소진되고, 나머지(185-170=15)는 재고로 남는다.
- `list_queue()`: 대기 중인 큐 항목을 FIFO 순서로 반환(현재 처리 대상 포함)
- 큐가 비어있을 때 `process_next()` 호출 시 None 반환(예외 아님 - 정상적인 "처리할 것 없음" 상태)

## 테스트 케이스

1. 부족분 170, 수율 0.92 → 실 생산량 ceil(170/0.92)=185, 총 생산시간 = 평균생산시간 * 185
2. 생산 완료 후 해당 주문 상태가 CONFIRMED로 전환되는지
3. 생산 완료 후 재고가 올바르게 반영되는지(주문 수량만큼 소진 가능한 재고 확보)
4. 큐가 비어있을 때 `process_next()`가 None을 반환하는지 (예외 발생 안 함)
5. 여러 건이 큐에 있을 때 FIFO 순서(먼저 등록된 것부터)로 처리되는지
