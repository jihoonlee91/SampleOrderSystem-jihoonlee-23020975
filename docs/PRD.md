# Product Requirements Document (PRD)

## 배경

가상의 반도체 회사 "S-Semi"는 다양한 반도체 시료(Sample)를 생산하여 연구소, 팹리스 업체, 대학 연구실에 납품한다.
시료는 주문이 들어오면 웨이퍼 공정 설비를 통해 제작되고 검수를 거쳐 고객에게 출고된다.
기존에는 엑셀과 메모장으로 주문을 관리하여 실수가 잦고 재고·공정 현황 파악이 어려웠다.
이를 해결하기 위해 콘솔 기반의 "반도체 시료 생산주문관리 시스템"을 개발한다.

## 역할과 흐름

- 고객: 필요한 시료를 이메일로 요청
- 주문 담당자: 요청에 맞게 주문서 작성(시료 예약)
- 생산 담당자: 개발 시료 등록, 주문 수신 후 승인 또는 거절

## 시스템 개요

- 콘솔 기반으로 동작하며 담당자가 직접 명령을 입력한다.
- 생산 라인은 하나이며, 한 번에 시료 하나씩 생산하고 주문이 들어온 시료만 생산한다.

## 주문 상태

| 상태 | 의미 |
|---|---|
| RESERVED | 주문 접수 |
| REJECTED | 주문 거절 (모니터링 제외) |
| PRODUCING | 승인 완료, 재고 부족으로 생산 중 |
| CONFIRMED | 승인 완료, 출고 대기 중 |
| RELEASE | 출고 완료 |

## 기능 요구사항

### 1. 시료 관리
- 시료 등록: 시료 ID, 이름, 평균 생산시간, 수율을 입력받아 새 시료 등록
- 시료 조회: 등록된 모든 시료 목록과 현재 재고 수량 표시
- 시료 검색: 이름 등 속성으로 특정 시료 검색

### 2. 시료 주문 (예약)
- 고객이 원하는 시료 ID, 고객명, 주문 수량을 입력받아 주문 생성
- 생성 시점의 주문 상태는 RESERVED

### 3. 주문 승인/거절
- RESERVED 상태 주문 목록 표시
- 승인 시:
  - 재고가 충분하면 즉시 CONFIRMED로 전환
  - 재고가 부족하면 생산 라인에 등록하고 PRODUCING으로 전환
- 거절 시: 즉시 REJECTED로 전환

### 4. 모니터링
- 주문량 확인: 상태별(RESERVED/CONFIRMED/PRODUCING/RELEASE) 주문 수 표시 (REJECTED 제외)
- 재고량 확인: 시료별 현재 재고와 주문 대비 상태(여유/부족/고갈) 표시

### 5. 생산 라인
- 부족분을 수율과 오차를 고려하여 생산: 실 생산량 = ceil(부족분 / 수율), 총 생산 시간 = 평균 생산시간 * 실 생산량
- 생산 완료 시 주문 상태를 PRODUCING → CONFIRMED로 변경
- 현재 생산 중인 시료 정보 표시
- 대기 중인 생산 큐를 FIFO 순서로 표시

### 6. 출고 처리
- CONFIRMED 상태 주문에 대해 출고 실행 시 RELEASE로 전환

## 비기능 요구사항

- 데이터 영속성: 프로세스를 재시작해도 등록된 시료/주문 데이터가 유지되어야 한다.
- 화면 구성(UI 레이아웃)은 자유롭게 결정 가능하다.

## 범위 밖 (Out of Scope)

- 실제 웹/GUI 인터페이스
- 다중 생산 라인 동시 운영
- 사용자 인증/권한 관리
